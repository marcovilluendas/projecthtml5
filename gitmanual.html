Manual básico de uso de GIT Git es un software de control de versiones utilizado para del mantenimiento de versiones en el desarrollo de aplicaciones. Se llama control de versiones a la gestión de las modificaciones realizadas en el código fuente de una aplicación. Para realizar este control se utilizan herramientas de gestión como Subversion o GIT. A continuación le mostramos las funciones básicas del uso del GIT. Recuerde que Para utilizar el cliente GIT en CDmon, primero deberá tener activado el acceso por SSH en el panel de control y tener habilitada la opción de GIT. Copiar el repositorio: Para utilizar el control de versiones deberemos tener un repositorio git creado en algún servidor remoto. Para conectar y copiar el contenido del repositorio podremos hacerlo con el comando clone git clone gitURL Ejemplo: git clone git@github.org:test/repositorio.git Esa orden creará el directorio que contendrá los ficheros. Añadir fichero al repositorio: Cuando tengamos un fichero listo para añadir al repositorio utilizaremos el comando add. git add fichero Ejemplo: git add test.html Una vez hecho esto no obtendremos respuesta del GIT, pero si efectuamos el comando status veremos que estará añadido git status Y mostrará lo siguiente: # On branch maste # Changes to be committed # (use "git reset HEAD ..." to unstage) # # new file: test.htm # El fichero esta añadido pero no se modificará el repositorio locals hasta que se realize el comando commit. git commit -m fichero Ejemplo: git commit -m test.html La respuesta será la siguiente: test[Mon Aug 20 15:43]# git commit -m test.html Created commit a9e19fb: test.html 1 files changed, 1 insertions(+), 0 deletions(-) create mode 100644 test.html No obstante este cambio no se habrá propagado a todos los repositorios. El commit sólo modifica el repositorio local. Para enviar al repositorio global se utiliza el comando push git push gitURL Ejemplo: git push git@github.org:test/repositorio.git Debemos de tener en cuenta que no todos los repositorios disponen de permisos para poder efectuar cambios. Tenemos que asegurarnos que disponemos de los permisos necesarios para publicar en el servidor donde tenemos el repositorio remoto. Si queremos "descargar" los ficheros que otros usuarios hayan actualizado podemos utilizar el comando pull git pull Algunos comandos de GIT: add: Añadir ficheros branch: Listar, Crear ramas de desarrollo clone: Clonar un repositorio a un directorio commit: Guardar los cambios al repositorio diff: Mostrar las diferencias entre versiones init: Crear un repositorio git vacio log: Mostrar el histórico de "commit" realizados merge: Unir dos o más ramas de desarrollo mv: Mover o renombrar un directorio pull: Recoger los ficheros de otro repositorio push: Enviar los dicheros al repositorio remoto reset: Deshacer los cambios y volver al estado anterior rm: Eliminar ficheros del árbol de desarrollo status: Mostrar el estado de la rama de trabajo actual Si desea hacernos algún comentario o sugerencia acerca de este tutorial no dude en ponerse en contacto con nosotros.
